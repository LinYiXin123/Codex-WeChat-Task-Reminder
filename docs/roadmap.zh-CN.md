# 路线图

这份路线图不按“功能数量”排序，而按“普通用户什么时候真的觉得这个项目有价值”排序。

当前北极星目标：

把本机 Codex 变成一个普通用户也能快速部署、稳定访问、愿意长期使用的浏览器工作台，尤其适合 Windows、Windows Server、手机浏览器和自托管远程访问。

## 第一阶段：5 分钟可用

目标：

- 让新用户第一次看到项目时，知道它是干什么的
- 让 Windows 用户一条命令就能跑起来
- 让安装完成后能立刻拿到访问地址、密码和重启命令

重点：

- Windows 一条命令安装继续稳定
- Cloudflare Tunnel 一键模式继续收口
- “让 Codex 自动部署”提示词继续优化
- 安装失败时输出更明确的排查建议
- README、发布页和截图继续围绕“快速可用”收口

## 第二阶段：手机端真的顺手

目标：

- 不是“能打开”就算完成，而是手机端真的能长期用

重点：

- 会话区长列表继续减卡顿
- 输入区、技能选择、下拉面板避免被输入法遮挡
- 会话切换、消息回显、加载状态继续减迟滞感
- 图片、本地文件、Markdown 链接展示更稳定
- 队列、引用立即执行、执行中状态继续贴近官方客户端体验

## 第三阶段：远程访问足够省事

目标：

- 没有公网 IP 的用户也能先用起来
- 有自托管经验的用户可以平滑升级到自己的入口方案

重点：

- Cloudflare Tunnel 快速隧道继续稳定
- 补 frp 服务端包和客户端示例
- 补 Tailscale / ZeroTier / Nginx / Caddy 示例
- 补远程访问安全清单
- 设置页显示当前远程入口状态和关键诊断信息

## 第四阶段：问题出现时能自己定位

目标：

- 用户不是只会说“卡住了”，而是能快速带着有效信息来反馈

重点：

- 服务健康检查更明确
- 日志目录、最近错误和 tunnel 状态更容易找到
- 导出诊断包，方便反馈安装和同步问题
- 常见故障自动提示修复命令
- 区分“前端假卡住”“同步延迟”“后端真异常”“公网链路异常”

## 第五阶段：成为默认推荐方案

目标：

- 当别人问“怎么把本机 Codex 放到手机上用”时，这个仓库能成为一个自然会被推荐的答案

重点：

- README 首屏继续强调定位，不再像内部说明书
- 发布页、截图、短视频和 social preview 持续完善
- GitHub topics、About、Issue 模板、路线图形成完整对外形象
- 把“Windows 友好 + 手机可用 + 自托管远程入口”打成清晰心智

## 暂不优先

这些方向不是永远不做，而是当前不该抢资源：

- 多用户权限系统
- 云端账号体系
- 重数据库依赖
- 大型团队协作功能
- 为了炫技增加复杂功能但牺牲部署门槛

原因很明确：

如果项目先失去“新手也能跑、手机也能用、Windows 也稳定”这三个基本盘，后面的高级能力不会带来真实增长。
